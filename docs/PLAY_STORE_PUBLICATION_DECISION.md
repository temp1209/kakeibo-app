# Play Store 公開方針

- 決定日: 2026-07-18
- 状態: 採用
- 対象: kakeibo-app

## 結論

現時点では Google Play Store への一般公開を行わない。

本アプリは自分自身の日常利用を主目的として継続し、ポートフォリオとしては README、スクリーンショット、デモ動画を公開成果物とする。Play Store 公開は、一般配布やストア公開そのものに明確な価値が生じた場合に改めて検討する。

## 背景

本アプリは、頻繁な手入力によって家計簿が続かなかった個人的な課題を解決するために開発した Android アプリである。レシート撮影から Gemini API による解析、分類、集計までを自動化し、実生活で継続利用できることを検証している。

主な目的は次の2点である。

1. 自分用の家計簿として実用する
2. 課題設定、設計、実装、実利用までを示すポートフォリオにする

不特定多数への配布や収益化は、現時点の主目的ではない。

## Play Store 公開に必要な負担

2026-07-18時点の Google 公式情報では、Play Console のデベロッパー登録料は一回限り 25 米ドルである。

また、2023-11-13以降に作成された新規の個人デベロッパーアカウントでは、本番公開を申請する前に次のクローズドテストが必要となる。

- 12人以上のテスターが参加する
- 各テスターが14日間連続でオプトイン状態を維持する
- 条件達成後、本番アクセスを申請し、テスト内容や公開準備に関する質問へ回答する

登録料に加え、テスターの募集、案内、参加状況の確認、フィードバック対応が必要になる。公開後もストア情報、ポリシー、データセーフティ、審査、アプリ更新への継続的な対応が発生する。

## 判断理由

### 1. 主用途に一般公開が必要ない

自分用アプリとしては、手元の端末へ直接インストールできれば目的を満たせる。Play Store を配布経路として用意しても、日常利用上の便益は限定的である。

### 2. 公開コストに対する効果が小さい

12人以上による14日間のテストは、実際に一般ユーザーへ提供する製品には有益である。一方、個人利用が中心の現状では、テスター確保と運営の負担が、得られる効果に対して大きい。

### 3. ポートフォリオの目的は別の成果物で達成できる

技術力やプロダクト設計を示すうえで重要なのは、ストア掲載の有無だけではない。次の情報があれば、課題と解決策、実装内容、動作実績を十分に説明できる。

- README: 背景、課題、設計意図、技術構成、実装済み機能
- スクリーンショット: 主要画面と完成度
- デモ動画: 起動、撮影、解析、一覧・分析への反映という一連の体験
- 実利用の記録: 自分自身が継続利用し、課題解決を検証した事実

特に本アプリの価値である「起動から少ない操作で記録できる体験」は、静的なストアページよりも短い動画の方が伝わりやすい。

### 4. 現在の配布モデルは一般利用向けではない

本アプリはバックエンドを持たず、利用者自身の Gemini API キーを端末内に設定する構成である。この方式は個人利用や技術デモには適しているが、一般ユーザー向けの導入体験としては説明と設定の負担が大きい。

## ポートフォリオとして用意するもの

### 必須

- 現行 README の維持・更新
- 主要画面のスクリーンショット4枚程度
  - カメラ・撮影
  - レシート一覧または詳細
  - 月次分析
  - 要確認・修正フロー
- 30〜90秒程度のデモ動画
  - アプリ起動
  - レシート撮影と送信
  - AI解析
  - 一覧または分析画面への反映

### 任意

- リリース用 APK の作成
- 面接やレビュー時に限った APK の個別共有
- アーキテクチャ図や設計判断の追記

APK を公開配布する場合は、署名、更新経路、API キーの扱い、第三者向けの利用説明を別途検討する。

## 再検討する条件

次のいずれかが生じた場合、Play Store 公開を再検討する。

- 不特定多数へ継続的に配布したい
- 実ユーザーから広くフィードバックを集めたい
- ストア審査・公開・運用の経験自体を実績にしたい
- API キーを利用者に設定させない提供方式を用意した
- テスター12人以上を無理なく確保できる
- 登録、テスト、審査、公開後の保守に時間を割く明確な理由がある

再検討時には、Google Play の要件が変更されている可能性があるため、最新の公式情報を確認する。
