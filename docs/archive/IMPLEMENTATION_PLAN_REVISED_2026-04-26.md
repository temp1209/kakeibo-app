## 次の実装計画（提案）

優先度は「ユーザー体感・要件ギャップが大きい順」を想定。

1. **一覧/詳細/通知の解析状態の可視化（Phase 5.2 残）**  
   同じ語彙で「解析待ち／実行中／完了／失敗／要確認」を揃え、一覧行・詳細ヘッダ・通知タブで迷わないようにする。

2. **Gemini JSON の閲覧体験（Phase 5.2 残）**  
   折りたたみ・コピー・可能ならシンタックス強調など、調査・サポート問い合わせ時の負担を下げる。

3. **Gemini の継続チューニング（Phase 5.1 残）**  
   実レシートに基づくプロンプト差分、回帰用フィクスチャの最小セットから着手。必要なら生レスポンス保存を検討。

4. **通知履歴の永続化（Phase 2.5 候補）**  
   要件の「見逃し対策」に直結。ローカルDBまたは DataStore で直近 N 件を保持する案が現実的。

5. **将来項目の切り出し**  
   オフライン撮影→後解析、プレビュー回転/トリミング、検索、再解析、バックアップ暗号化、Sign-In の Credential Manager 移行などは、上記が安定してからマイルストーンを分割する。
