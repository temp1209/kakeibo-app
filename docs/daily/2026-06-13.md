## 実施したこと

### 1. 開発用データ退避（リリース機能にはしない）
- 改善着手前のバックアップ要望を受け、当初は設定画面から JSON ファイル出力を検討

### 2. 開発環境の再構築（C ドライブ初期化後）
- Android Studio + JBR は `C:\Program Files\Android\Android Studio\jbr` に存在することを確認
- ターミナルで `gradlew` 実行時の **`JAVA_HOME is not set`** を、ユーザー環境変数 `JAVA_HOME` の設定で解消
- `adb` は SDK `platform-tools` を PATH に追加（またはスクリプト内で `%LOCALAPPDATA%\Android\Sdk` から自動検出）

### 3. 退避スクリプトの修正・実行
- PowerShell の引用符/`&&` 解釈によるパースエラーを修正（`cmd /c` 経由、adb パス自動検出）
- 実行ポリシーで止まる場合は `-ExecutionPolicy Bypass` を使用
- `backups/dev/2026-06-13_210431/kakeibo.db` を取得（約 455 KB、SQLite ヘッダー正常）
- DB Browser for SQLite で中身を確認（テキストエディタでの文字化けはバイナリのため正常）

## 結果・所見

- 改善前の保険として **PC 上の SQLite バックアップ + Drive JSON バックアップ** の二段構えが現実的
- 新 PC では **debug.keystore の SHA-1 が変わる**ため、Drive ログイン前に Google Cloud Console へ SHA-1 追加が必要（再構築時の定番）
- 退避スクリプトは debug ビルド専用。本番 APK では `run-as` 不可
