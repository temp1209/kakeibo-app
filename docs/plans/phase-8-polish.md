## 3. サブフェーズ

### 8.1 Gemini JSON 閲覧改善（Phase 5.2 残）

**現状**（`ReceiptDetailScreen.kt`）:

- メニュー「Gemini JSON」→ `AlertDialog` 内に **1行の minify JSON** を等幅フォント表示
- 長文はスクロール可能だが読みにくい。コピー手段なし

**改修内容**:

| 項目 | 内容 |
|------|------|
| 表示コンテナ | `AlertDialog` → `ModalBottomSheet`（高さ `fillMaxHeight(0.85f)` 程度） |
| 整形 | `rawJson` を pretty-print（`JSONObject` / Gson `JsonParser` で indent 2） |
| コピー | ツールバーに「コピー」→ `ClipboardManager` + Snackbar「コピーしました」 |
| 折りたたみ | **初版は不要**。pretty-print で十分なら見送り（必要なら `items` セクションのみ折りたたみを 8.1+ で検討） |
| 共通化 | `ui/common/GeminiJsonViewerSheet.kt`（詳細画面から呼ぶ） |

**対象ファイル**: `ReceiptDetailScreen.kt`（新規 `GeminiJsonViewerSheet.kt`）

**完了条件**:

- [ ] 複数行インデントで表示される
- [ ] コピーでクリップボードに全文が入る
- [ ] JSON なしのときメニューは従来どおり無効

**規模**: S（半日以内）

### 8.2 オンボーディング UI ブラッシュアップ（Phase 7.1 後回し）

**現状**: 機能は実機確認済み。ステップインジケータ・余白・文言が最小実装のまま。

**改修内容**（最小セット）:

| 項目 | 内容 |
|------|------|
| ステップ表示 | 画面上部に「2 / 5」のようなインジケータ（API 33 未満は通知ステップ省略時は分母を 4 に） |
| レイアウト | 各ステップの `padding`・タイトル/本文の typography を統一 |
| 文言 | 権限ステップの説明を1行補足（カメラ＝撮影、通知＝解析完了のお知らせ） |
| アニメーション | **任意**。`AnimatedContent` でステップ切替が軽ければ追加、なければ見送り |

**スコープ外**: イラスト追加、長いチュートリアル、再表示 UI

**対象ファイル**: `OnboardingWizard.kt`, `OnboardingNavRow.kt` 等

**完了条件**:

- [ ] 新規インストールでステップ位置が分かる
- [ ] 既存の自動遷移・スキップ・APIキー次へボタン挙動は維持

**規模**: S〜M（0.5〜1 セッション）

### 8.3 オンボーディング技術改善（§13 中程度）

[`onboarding.md`](onboarding.md) §13 の **M1 / M2** を解消する。

| ID | 問題 | 対策 |
|----|------|------|
| **M1** | 設定アプリで許可後に戻ると UI が未許可のまま | `OnboardingPermissionStep` で `Lifecycle.Event.ON_RESUME` 時に `checkSelfPermission` 再評価 |
| **M2** | 許可時に `onNext()` が二重呼び出しされうる | 自動遷移は `LaunchedEffect(granted)` のみ。`onResult` は `granted` 状態更新だけ |

**完了条件**:

- [ ] 設定から戻ったあと許可済み表示になる（または自動で次へ）
- [ ] 許可直後のステップ飛ばし・ループが再発しない

**規模**: S（0.25〜0.5 セッション）
