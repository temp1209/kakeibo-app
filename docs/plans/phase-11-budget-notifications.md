## 1. 背景と目的

| 柱 | 内容 |
|----|------|
| **通知** | マスター + 個別トグル。失敗のみデフォルト ON |
| **月次予算** | オンボーディング導入、10/20/月末通知 |
| **分析 UI** | 棒全体 = 予算、必須/裁量を積み上げ |

## 2. 通知

### 2.1 種別とデフォルト

| 種別 | デフォルト | 設定 |
|------|------------|------|
| すべての通知（マスター） | ON | ✅ |
| 解析失敗 | 権限許可時 **のみ ON** | ✅ |
| 解析完了 | OFF | ✅ |
| 要確認（低信頼） | OFF | ✅ |
| 月次予算（10/20/月末 + 閾値） | OFF | ✅ |

### 2.2 オンボーディング（通知ステップ・新規のみ）

- 権限 **許可** → `解析失敗 = ON`、他 OFF
- 権限 **拒否** → すべて OFF
- 文言: 失敗時のお知らせ + 予算ペース通知の説明（完了通知は推さない）

### 2.3 設定 UI

```
[ すべての通知 ]

  解析が失敗したとき
  解析が完了したとき
  要確認のレシートがあるとき
  月次予算の進捗（10日・20日・月末）
```

- マスター OFF → Worker / `AnalysisNotifications` で送信しない
- 予算無効時は「月次予算」トグル非表示

### 2.4 月次予算通知のスケジュール

| タイミング | 内容 |
|------------|------|
| 毎月 **10 日** | 予算に対する使用割合（%） |
| 毎月 **20 日** | 同上 |
| **月末**（最終日） | 同上 + 月間使用総額 |
| 閾値 80% / 100% | 到達時（個別 ON/OFF 可） |

日付は **固定**（ユーザーカスタムなし）。`WorkManager` + 月初/日次チェックで実装。

## 3. 月次予算

### 3.1 オンボーディング（新規のみ）

フロー:

```
① ようこそ → ② APIキー → ③ 月次予算 → ④ カメラ → ⑤ 通知 → ⑥ 完了
```

- ③: 機能紹介 + 月額（円）入力。**スキップ可** → `budgetEnabled = false`
- 既存ユーザー（`onboardingCompleted` 済み）には **再表示しない**。設定から有効化

### 3.2 データ

| フィールド | 説明 |
|------------|------|
| `budgetEnabled` | 予算機能 ON/OFF |
| `monthlyBudgetYen` | 月額予算（円） |
| `budgetAggregateMode` | `TOTAL`（必須+裁量） / `DISCRETIONARY_ONLY` |

- バックアップ 1.2 に `budget` セクション追加（schema bump 要検討）

### 3.3 集計

- `receipt_items`（`isAdjustment=0`）、当月（`receiptDatetime` / `capturedAt`）
- `TOTAL`: 必須寄り + 裁量寄りの合計金額
- `DISCRETIONARY_ONLY`: `necessityScore < 50` の行のみ

## 4. 分析タブ UI

### 4.1 予算有効時

棒の **全幅 = 月次予算**。左から必須 → 裁量を積み上げ、右が残り。

```
予算 ¥80,000
[■■必須■■][■裁量■][    残り    ]
使用 ¥57,000（71%）
```

- 超過: 棒満杯 + `予算超過 ¥X` テキスト
- `DISCRETIONARY_ONLY`: 棒は予算、充填は裁量のみ。必須は参考ラベル

### 4.2 予算無効時

- 支出のみの積み上げ棒（幅 = 支出 100%）
- 設定への短い導線

### 4.3 実装

- Compose 自前（`Row` + `weight`）。チャートライブラリ不要
- 現行 `LinearProgressIndicator` を置換

## 5. 解析失敗 UX（Phase 10 補完）

| 項目 | 内容 |
|------|------|

| 詳細 | 全文エラー + 再送信ボタン |
| 通知 | 失敗のみ（設定連動）。アプリ内が主 |

`GeminiUserMessages` → プロバイダ非依存の `AiUserMessages` へリネーム検討（Phase 10 と同時でも可）。
