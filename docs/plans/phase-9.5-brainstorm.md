## 2b. ブレスト時のスコープ案（アーカイブ・参照用）

> 以下は分割前のたたき台。実装スコープは §2 と Phase 10/11 ドキュメントを正とする。

### 柱 1: 解析失敗体験（→ Phase 10 + 11 に分割）

**ゴール**: 失敗しても「何が起きたか・次に何すればいいか」がアプリ内で完結する。OS 通知に頼らない。

| 項目 | 内容 |
|------|------|
| 一覧・詳細での失敗表示 | 理由を `GeminiUserMessages` ベースで常時表示（レート制限は明示） |
| 再送信導線 | 失敗レシートからワンタップ再解析（既存手動再送信の強化） |
| 上限時の案内 | 「しばらく待つ」に加え、無料枠の性質・確認先（Google AI Studio 等）へのリンクは任意 |
| キュー状態 | 解析中 / 失敗 / 要確認を一覧で一目で（5.2 の延長） |
| 通知との分離 | 失敗は **アプリ内で十分**。解析完了 OS 通知は別途オフ可能に（柱 2） |

**スコープ外**: 自動リトライ（バックオフ Worker）は初版は見送り可。ユーザー操作の再送信を明確にする方が先。

### 柱 2: 通知の整理

| 通知種別 | デフォルト | 設定で個別 ON/OFF |
|----------|------------|-------------------|
| **解析完了** | OFF | ✅ |
| **解析失敗** | 権限許可時のみ ON（下記） | ✅ |
| **要確認（低信頼）** | OFF | ✅ |
| **月次予算**（10/20/月末 + 閾値） | OFF（予算有効時も初期 OFF） | ✅ |
| **すべての通知**（マスター） | ON（ただし種別ごと初期は上表） | ✅ |

#### オンボーディング（通知権限ステップ）

- ユーザーが **通知権限を許可したときだけ** → **解析失敗通知のみ ON** にする
- 許可しなかった／スキップした → 種別はすべて OFF（マスターは ON のままでも実質送信なし）
- 文言: 「解析完了」ではなく **失敗時のお知らせ** と予算ペース通知の説明に更新（完了通知は推さない）

#### 設定 UI（通知セクション）

```
[ すべての通知 ]  ── マスタートグル（OFF で個別トグル無効化 or 一括 OFF）

  解析が失敗したとき
  解析が完了したとき
  要確認のレシートがあるとき
  月次予算の進捗（10日・20日・月末）
```

- マスター OFF → OS 通知は一切出さない（Worker 側でもガード）
- マスター ON → 個別トグルに従う
- 予算機能無効時は「月次予算」トグルは非表示 or 無効

**実装メモ**: `NotificationPrefs`（DataStore or SharedPreferences）にマスター + 種別フラグ。既存ユーザーは移行時 **すべて OFF**（失敗のみ ON にしたい場合は設定で明示ON）

### 柱 3: 月次全体予算 + 消費ペース

**ゴール**: コーチではなく **鏡**。数字と割合だけ伝える。

| 項目 | 内容 |
|------|------|
| **オンボーディング** | **APIキー直後**に予算ステップを追加。紹介 + 月額入力を促す。**スキップ可** → **予算機能無効**。**新規のみ** |
| 月の全体予算 | 1つだけ（円） |
| **集計対象（設定）** | デフォルト: **支出合計（必須+裁量）**。切替: **裁量のみ** |
| 分析タブ表示 | **棒全体 = 月次予算**。その中に必須・裁量を **左から積み上げ**。未使用分は空き |
| 予算無効時 | 予算枠なしの **支出のみ積み上げ棒**（幅 = 必須+裁量）。設定で予算有効化を促す |
| **定期通知** | 月次リズムに合わせ **10日・20日・月末**（固定。個別カスタムはしない） |
| 閾値通知 | 例: 80% / 100% 到達時（ON/OFF 可） |

**集計**: `receipt_items` 調整行除外。対象は上記設定に従う。

**分析タブ UI（合意）**

**予算有効時** — 棒の全幅が月次予算。支出はその中に必須→裁量の順で積み上げる:

```
予算 ¥80,000
[■■必須 ¥45k■■][■裁量 ¥12k■][      残り ¥23k      ]
 ←──────────── 棒全体 = 予算 100% ────────────→

使用 ¥57,000（71%）　必須 32行 / 裁量 18行
```

- **棒全体の幅** = 月次予算（100%）
- **左から積み上げ**: 必須寄り（≥50）→ 裁量寄り（<50）。各セグメント幅 = その金額 ÷ 予算
- **右側の空き** = 予算残り（未使用）
- **予算超過時**: 棒は満杯表示 + テキストで超過額（例: `予算超過 ¥5,000`）。セグメントは予算内に収まるようクリップ or 超過分を別色で右端に示す（実装時に決定）
- 色: Material3 で必須 / 裁量 / 残り（surfaceVariant）を区別

**集計軸が「裁量のみ」のとき**

- 棒全体は依然 **予算**
- 積み上げは **裁量分のみ**（必須は棒外に参考表示するか、非表示 — 実装時は参考ラベルのみ推奨）
- 使用率・通知も裁量 ÷ 予算

**予算無効時**

- 予算枠のない横向き積み上げ棒（幅 = 支出合計 100%、必須 + 裁量）
- 「予算を設定すると、予算に対する進捗が見えます」等の短い導線（設定へ）

**スコープ外**: チャートライブラリ導入（Compose `Row` + `Box` + `weight` で自前）

**オンボーディングフロー（合意）**

```
① ようこそ → ② APIキー → ③ 月次予算（新規） → ④ カメラ → ⑤ 通知 → ⑥ 完了
```

- ③ スキップ → `budgetEnabled = false`。後から設定で有効化
- ⑤ 許可 → **解析失敗のみ ON**。拒否 → すべて OFF

**スコープ外**: ジャンル別予算、通知日の自由設定、AI 節約アドバイス、チャートライブラリ導入（Compose 自前で十分）

## 6. カスタム性の方針（個人利用 × Play 公開）

### 前提

- **作り手**: 自分が使いやすいもの（個人最適）
- **公開**: ポートフォリオ + Play Store 想定 → **破綻しないデフォルト** と **説明可能な範囲** は必要
- **原則**: カスタムは **機能の意味に直結するもの** だけ。見た目・好みの細分化は最小限

### 3 層モデル（労力の切り方）

| 層 | 意味 | 例 | 方針 |
|----|------|-----|------|
| **A. 意味のある設定** | 家計の見え方・通知の中身が変わる | 予算額、集計対象（合計/裁量）、必須度方針プリセット、解析完了通知 ON/OFF | **実装する**。Play 公開でも説明しやすい |
| **B. 固定の良いデフォルト** | 好みは割れるが、選ばせなくても困らない | 予算通知日（10/20/月末）、閾値 80%/100%、Material 動的色 | **固定 or 2〜3択まで**。個別カスタム UI は作らない |
| **C. 装飾・嗜好** | 見た目・マイナーな頻度 | テーマカラー、フォント、通知の曜日指定 | **原則やらない**。ダークモードは OS 追従で足りる |

### 判断のショートカット

新しい「設定項目」を足す前に:

1. **スキップ or デフォルトで困るユーザーがいるか？** → No なら設定化しない（B 層で固定）
2. **設定を間違えると家計の数字がおかしくなるか？** → Yes ならオンボーディング or 説明文を厚く（A 層）
3. **ポートフォリオで語れるか？** → 「撮影中心の家計可視化」に関係ないなら優先度下げ（C 層）

### 今回の合意との対応

| 項目 | 層 | 決定 |
|------|-----|------|
| 月額予算・集計対象 | A | オンボーディング + 設定 |
| 予算通知 10/20/月末 | B | **固定**（設定画面に説明のみ） |
| 解析完了通知 | A | ON/OFF（**デフォルト OFF**） |
| 解析失敗通知 | A | 権限許可時のみ ON、設定で変更可 |
| 通知マスタートグル | A | すべての通知 ON/OFF |
| テーマカラー | C | **やらない**（Material3 + システムテーマ） |
| 通知頻度の自由入力 | C | **やらない** |

### Play Store / ポートフォリオ向けの言い方

- 「自分用に作ったが、**レシート撮影で家計が見える**汎用アプリとして公開」
- カスタムは **方針・予算・通知の要否** に絞り、**好みの細かいチューニングはしない** と明記できる
- スクリーンショットは **デフォルト体験** 前提で撮る（設定の組み合わせ爆発を避ける）
