## 1. アプリのコア（変えない軸）

| 軸 | 内容 |
|----|------|
| **入力** | レシート撮影が主。手入力は例外 |
| **価値** | 無駄遣い（裁量支出）の可視化 |
| **AI の役割** | OCR・分類・必須度付与。ユーザーは方針と訂正例でチューニング |
| **スコープ** | 個人利用。複雑な家計簿機能にはしない |

### Phase 9 までで得たもの

- ユーザーごとの必須／裁量の「方針」
- 訂正例 → コンパイル → 解析・再スコアへの反映
- プリセット別のスコア帯・境界ケース

### まだ弱いところ（仮説）

- 方針を変えても **「今月どうなってる？」** が一目で刺さらない
- レシートが増えると **探す・振り返る** がしづらい
- プロンプト改善の恩恵が **過去レシートに届きにくい**
- 分析は月次サマリ＋TOP リスト止まり

## 3. コンセプトの伸びしろ（問い）

- **誰のため？** 節約したい人 / 家計を把握したい人 / 記録が面倒な人 — どこを主ターゲットにするか
- **いつ刺さる？** 月末振り返り / 買い物直後 / レシート撮った直後 — 通知・分析の設計に効く
- **何と差別化？** 一般的な家計簿アプリは手入力・予算管理。本アプリは **撮影 + 裁量の自動判定**
- **AI コストの上限** — 月いくらまで許容？ それが再解析・再スコア頻度の設計を決める
