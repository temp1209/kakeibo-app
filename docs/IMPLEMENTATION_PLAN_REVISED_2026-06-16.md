### 7.3 API キー未設定時の送信フィードバック（最優先・小〜中）

**背景**: プレビューで送信確定しても API キー未設定の旨が表示されず、`AnalysisWorker` がサイレントスキップする。6.2 の再送信には `ApiKeyMissing` ガードがあるが、**初回送信には未適用**。

- [ ] **7.3.1 プレビュー送信前ガード**
  - `PreviewScreen` または `MainActivity` で `GeminiApiKeyStore.hasKey()` をチェック
  - 未設定時: ダイアログ「APIキーが必要」+ 設定タブへ遷移（`navigateToTabRoot(Route.Settings)`）
- [ ] **7.3.2 Worker の二重防御**
  - キー未設定でキュー処理時はレシートを `FAILED` + `analysisErrorMessage`（「APIキー未設定」）に更新
  - `AnalysisWorker` の `readKeyOrNull() ?: return Result.success()` を廃止
- [ ] **7.3.3 文言統一**
  - 6.2 再送信の `ApiKeyMissing` メッセージと同一文案

**完了条件**: APIキー未設定で送信確定しようとすると理由が表示され、設定へ誘導される。

**関連コード**: `PreviewScreen.kt`, `MainActivity.kt`, `AnalysisWorker.kt`, `GeminiApiKeyStore.kt`

---

### 7.2 Drive バックアップのパーミッションエラー（高優先・中）

**背景**: ログインは成功するがバックアップ API が 403 等で失敗（開発環境再構築後に顕在化）。SHA-1 不一致が有力候補。

- [ ] **7.2.1 環境要因の切り分け（手動・ドキュメント済み）**
  - 現行 debug.keystore の SHA-1 と Google Cloud OAuth クライアントの一致確認
  - `EXTERNAL_SETUP.md` §6、`DEBUGGING_GUIDE.md` §8 に手順あり
- [ ] **7.2.2 エラー表示の改善（コード）**
  - `DriveBackupWorker` の 403 を `Result.success()` で握りつぶさない
  - 設定画面 Snackbar に HTTP ステータスと対処ヒント（SHA-1 再登録案内）
  - Logcat に `DriveHttpException` 本文を残す
- [ ] **7.2.3 再ログイン導線**
  - スコープ不足時はサインアウト → 再サインインで `drive.appdata` を再要求

**完了条件**: 「今すぐバックアップ」が成功する、または失敗時に具体的な対処が UI に表示される。

**関連コード**: `GoogleSignInHelper.kt`, `DriveBackupOrchestrator.kt`, `DriveBackupWorker.kt`, `SettingsScreen.kt`
