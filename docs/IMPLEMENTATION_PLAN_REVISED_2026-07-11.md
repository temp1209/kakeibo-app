| 判断 | 理由 |

| **Drive 自動バックアップを廃止** | 個人利用では OAuth / 403・404 / manifest 運用コストがメリットを上回る。開発停滞の主因だった |

## 全体方針（2026-07-11 時点）

1. **コア体験を主役に戻す** — 撮影 → 解析 → 一覧/分析。保全は「動く手動バックアップ」で十分。
2. **未コミットの 7.2' を区切ってコミット** — 実機スモーク後に `main` へ反映。
3. **初回体験の体系化** — 7.1 オンボーディング実装済み。実機確認後 Phase 5 残へ。
4. **Phase 5 残は 7.1 の後** — 解析状態可視化は 7.3 の `FAILED` 表示と語彙統一で相乗。
5. **ドキュメントを現状に同期** — 要件・既知課題・外部設定から Drive 前提を除去または「過去」へ。
